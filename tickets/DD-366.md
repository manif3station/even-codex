# DD-366

## Title

Rework the Even plugin connector flow around DD helper auth and DD API-key auth.

## Method

BDD

## Acceptance

- the DD-served Even plugin clearly supports two DD connector modes over HTTPS:
  - helper login through the DD page session
  - API-key login using a DD username-style key id plus secret
- the plugin stores the connector base as one DD HTTPS ajax base such as
  `https://192.168.1.20:7890/ajax/even-codex`
- the plugin fetch layer sends `X-DD-API-Key` and `X-DD-API-Secret` only when
  one connector profile is in machine-auth mode
- the phone-side plugin UI explains which mode is active and what DD setup the
  operator must do first:
  - `dashboard auth add-user ...` for helper mode
  - one configured DD API client for machine-auth mode
- the plugin keeps transcript reads and prompt writes on the same DD connector
  endpoints for both modes
- source and browser tests in `t/` prove:
  - connector profiles persist auth mode and only attach API headers when
    configured to do so
  - helper mode keeps using the existing browser session flow
  - machine-auth mode uses the DD API headers on bootstrap, session, and prompt
    requests
- Docker tests and coverage pass at `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `even-codex` repo
