# DD-365

## Title

Add production-safe DD machine-auth support for the governed ajax routes.

## Method

TDD

## Acceptance

- `even-codex` uses DD's native layered `config/api.json` machine-auth contract
  without shipping a fake shared API client in the skill repo
- runtime DD `config/api.json` entries can authorize only the governed
  `even-codex` connector surface:
  `/ajax/even-codex/bootstrap`, `/ajax/even-codex/session`, and
  `/ajax/even-codex/prompt`
- helper-session auth still works on those same routes
- wrong or missing machine credentials for those registered routes fail with
  `403 {"status":"forbidden"}`
- the saved ajax handlers keep their current paired-session behavior once
  machine auth succeeds
- source and runtime tests in `t/` prove:
  - the installed DD route payloads still advertise the governed connector URLs
  - machine-auth requests can read bootstrap and session payloads and can submit
    prompts through the DD connector
- Docker tests and coverage pass at `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `even-codex` repo
